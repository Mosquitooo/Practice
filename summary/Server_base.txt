
1. 网络模型

	搞笑的事件处理模式
	reactor：主线程负责监听. 工作线程负责接收连接、读写数据、逻辑处理.
	
	proactor: 主线程处理所有的IO操作，工作线程只负责业务逻辑。
	
	
	高效的并发模式
	
	半同步半反应堆: 单进程多线程，主线程处理IO，工作线程处理逻辑
		缺点：
			1. 所有线程共享消息队列，任何操作都需要对队列加锁，浪费CPU时间。
			2. 每个工作线程同一时间只能处理一个请求，如果请求较多，工作队列少，
				客户端响应会越来越慢，如果增加工作线程，线程间切换也会耗费大量CPU时间
	
	半同步半异步：单进程多线程，异步线程处理IO事件，同步线程处理客户逻辑.
		解决了半反应堆的缺点. 主线程只管监听，工作线程负责连接，收发，处理
	
	领导者追随者：单进程多线程，请求到达时，领导者先获取请求，之后选出新的领导者

3. 三次握手与四次挥手
	三次握手：	
	client                           Server
	connect阻塞       SYN->          
	                <-SYN+ACK        accpet阻塞
	connect返回       ACK->          accpet返回

	四次挥手：	
	client                           Server
				FIN->
				<-ACK
				<-FIN
				ACK->
